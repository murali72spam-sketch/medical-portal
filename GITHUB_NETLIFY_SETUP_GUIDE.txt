# GitHub Pages Medical Portal - Setup & Launch Guide

**Complete step-by-step instructions to get your portal live on Netlify**

---

## Step 1: Create a GitHub Account (If You Don't Have One)

1. Go to [github.com](https://github.com)
2. Click "Sign up"
3. Create account with:
   - Email address
   - Password
   - Username (e.g., `muraligopal-dr`)
4. Verify email
5. Complete setup wizard

---

## Step 2: Create a GitHub Repository

1. **Log in to GitHub**
2. **Click "+" icon** (top right) → "New repository"
3. **Fill in:**
   - Repository name: `medical-portal` (or your preferred name)
   - Description: "Dr Murali Gopal's Paediatric Health Portal"
   - Public (so it can be deployed)
   - **Check:** "Add a README file"
4. **Click "Create repository"**

---

## Step 3: Upload Your Portal Files

### Option A: Using GitHub Web Interface (Easiest)

1. **Go to your new repository**
2. **Click "Add file"** → "Upload files"
3. **Drag & drop** the entire `medical-portal` folder contents
   - Upload: `index.html`, `css/`, `js/`, `education/`, `symptoms/`, etc.
4. **Scroll down** → "Commit changes" → "Commit directly to main branch"
5. **Done!** Files are now in GitHub

### Option B: Using Git Command Line (For Developers)

```bash
# Navigate to your project
cd medical-portal

# Initialize git
git init

# Add all files
git add .

# Commit changes
git commit -m "Initial commit: Medical portal MVP"

# Add GitHub as remote (replace with your repo URL)
git remote add origin https://github.com/yourusername/medical-portal.git

# Push to GitHub
git branch -M main
git push -u origin main
```

---

## Step 4: Set Up Netlify Deployment

### 4a: Create a Netlify Account

1. Go to [netlify.com](https://netlify.com)
2. Click "Sign up"
3. **Choose "GitHub"** (easiest option)
4. **Authorize Netlify** to access your GitHub account
5. Complete signup

### 4b: Deploy Your Repository

1. **In Netlify Dashboard**, click **"New site from Git"**
2. **Select GitHub** as your Git provider
3. **Search for your repository** (`medical-portal`)
4. **Click to select it**
5. **Configure build settings:**
   - Build command: (leave empty)
   - Publish directory: `.` (the root folder)
6. **Click "Deploy site"**

Netlify will now:
- Build your site
- Deploy to their servers
- Give you a live URL (e.g., `https://abc123.netlify.app`)

**Your site is now LIVE!** 🎉

---

## Step 5: Configure Custom Domain (Optional)

Once deployed, you can add a custom domain:

1. **In Netlify**, go to **Site settings** → **Domain management**
2. **Click "Add custom domain"**
3. **Enter your domain** (e.g., `pediatricportal.in`)
4. **Follow DNS setup instructions**
   - This depends on where you bought the domain
   - Typically takes 24-48 hours to activate

For now, use the Netlify domain (e.g., `medical-portal.netlify.app`)

---

## Step 6: Update & Deploy Changes

### Making Updates is Super Easy

1. **Edit a file** (e.g., add a new article)
   - Example: Create `education/fever.html`

2. **Push to GitHub:**
   ```bash
   git add education/fever.html
   git commit -m "Add article: Fever in children"
   git push origin main
   ```

3. **Netlify automatically:**
   - Detects the change
   - Redeploys your site
   - Updates live within 1-2 minutes

**No manual deployment needed!** 🚀

---

## Step 7: Test Your Live Site

1. **Go to your Netlify URL** (from Step 4b)
2. **Test on multiple devices:**
   - Desktop browser
   - Mobile phone
   - Tablet
3. **Check:**
   - All links work
   - Images load
   - Mobile menu works
   - Print preview looks good

## Common Tasks After Launch

### Adding a New Article

1. Create file: `education/my-article.html`
   - Copy from `education/template-article.html`
   - Edit content

2. Update `education/index.html` with link to new article

3. Push to GitHub:
   ```bash

   git commit -m "Add article: [Article Title]"
   ```

4. **Done!** Live in 1-2 minutes

## Troubleshooting

### Site Not Showing?

**Q: I see "Page not found" on Netlify**
- A: Check that files are uploaded to GitHub
- Go to your GitHub repo → files should be visible
- If not there, upload them (see Step 3)

**Q: Netlify says "Deploy failed"**
- A: Check the build logs in Netlify dashboard
- Usually just means a file is missing
- Verify all files are in GitHub repo

### Links Broken?

**Q: Links to articles don't work**
- A: Check file paths in links
  - Correct: `href="education/fever.html"`
  - Wrong: `href="/education/fever.html"` (extra `/`)

**Q: CSS not loading**
- A: Same as above - check paths
  - Correct: `<link rel="stylesheet" href="css/style.css">`

### Mobile Not Working?

**Q: Mobile menu doesn't open**
- A: Netlify might be serving old version
- Hard refresh: `Ctrl+Shift+R` (or `Cmd+Shift+R` on Mac)
- If still broken, check that `js/main.js` is in GitHub
