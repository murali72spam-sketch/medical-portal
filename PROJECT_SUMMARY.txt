## How to Use This (Step-by-Step)

### Phase 1: Setup (Day 1)
1. **Download** the `medical-portal` folder
2. **Create GitHub account** at github.com
3. **Create repository** named `medical-portal`
4. **Upload** all files to GitHub
5. **Create Netlify account** at netlify.com
6. **Connect** GitHub repo to Netlify
7. ✅ **Your site is LIVE!**

### Phase 2: Customization (Days 2-3)
1. **Edit** clinic name & location in index.html
2. **Update** colors if desired (css/style.css)
3. **Test** locally before pushing
4. **Push** changes to GitHub
5. **Verify** live site updated

### Phase 3: Content (Ongoing)
1. **Copy** article template → education/
2. **Fill** content sections
3. **Add** link to education/index.html
4. **Test** locally
5. **Push** to GitHub
6. ✅ **Live in 1-2 minutes!**

## Important Notes

### Medical Content
- ✅ Link to authoritative sources (IAP, WHO)
- ✅ Use evidence-based information
- ✅ Always include disclaimers
- ✅ Update when guidelines change
- ❌ Never invent medical facts
- ❌ Never diagnose patients

### Maintenance
- **Monthly:** Review 1-2 articles, add 1 new article
- **Quarterly:** Check mobile, update links
- **Yearly:** Full audit, consider redesign

### Git/GitHub Tips
- **Always test locally** before pushing
- **Write clear commit messages** (helps you remember)
- **Push frequently** (daily or weekly, not monthly)
- **Keep GitHub repo PUBLIC** (so Netlify can deploy)
